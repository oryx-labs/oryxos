# OryxOS 是什么

OryxOS 是基于 Java 21 构建的开源**分布式 AI Agent OS**。一份配置定义一个 Agent，一个底座运行一群 Agent。私有部署在你自己的 K8s 或服务器上，让一群业务 Agent 像进程跑在操作系统上一样，可靠地运行和协同。

## Agent OS 与 Agent Runtime 的区别

Agent 生态里已经有成熟的 runtime，OryxOS 做的是不同的事。

**Agent Runtime** 是让单个 Agent 跑起来的执行内核：调用模型、执行工具、管理上下文、控制推理循环。

**Agent OS** 在 runtime 之上，管理的是一群 Agent：多个 Agent 的生命周期、统一的对外渠道、统一的记忆、多租户与治理，以及分布式形态下的跨节点协作。

借操作系统类比——runtime 像单个进程的执行环境，Agent OS 像管理一群进程、调度资源、提供共享服务的那一层。runtime 让一个 Agent 跑起来，Agent OS 让一群 Agent 被运行和管理起来。

**OryxOS 是后者。**

## 为什么需要 OryxOS

Agent 生态里已有成熟框架，但几乎全部构建在 Python 和云原生技术栈上。对于后端事实标准是 Java、又出于合规和自主可控需要私有部署的企业来说，Java 生态里至今没有原生、可私有部署、开箱即用的 Agent OS。OryxOS 填补了这个空白。

更深一层的判断是：让 Agent 在生产环境可靠工作，瓶颈通常不在模型本身，而在 Agent 的运行环境。一个 Agent 能不能真正干活，取决于它有没有可靠的运行底座——能拿到对的上下文、有受控的工具、调用能被隔离和审计、跨节点协作时消息能可靠送达。OryxOS 做的不是又一个 Agent，而是这个让一群 Agent 可靠运行和协同的底座本身。

## 五大核心能力

**对接 LLM**
通过 Provider 抽象统一对接主流大模型，Agent 不感知具体厂商，运行时切换无锁定，支持本地推理（Ollama）。多 Provider 并存通过显式映射区分，不靠类型扫描。

**ReAct 循环**
自己实现的推理引擎，不套外部 Agent 框架。LLM 决定是否调工具、调哪个；OryxOS 执行后回填结果；LLM 再决定下一步，直到给出最终响应或达到最大迭代次数。循环行为完全可控。

**记忆**
让 Agent 跨对话保留状态。会话记忆 + 长期记忆（`MEMORY.md` 关键词检索），长期记忆自动注入每次 system prompt。接口预留向量检索升级空间。

**工具体系**
内置文件、Shell、HTTP 等基础工具。扩展三档接入，门槛从低到高：

- 零代码：写 `SKILL.md` 接入现成 MCP server
- 轻代码：用任意语言自写 MCP server
- 重代码：注册 Spring `@Tool` Bean，进程内直接调用

**对外服务**
所有能力通过 REST API 对外暴露，业务系统 HTTP 接入，任何语言均可集成。

## 设计原则

- **底座优先于 Agent** — 最重要的交付是让任意 Agent 都能可靠运行的环境，而不是某个强大的 Agent
- **自实现核心，可控优先** — 核心推理循环自己实现，协议适配复用成熟库，不重复造轮子
- **配置即 Agent** — 一个 Agent 由一份配置定义，而不是由代码写出
- **对接开放标准** — 工具用 MCP，协作用 A2A，技能用开放格式，与生态协同
- **无状态实例，状态外置** — 从单机平滑走向分布式的前提
- **安全是地基不是补丁** — 工具来源受控、最小权限、强制沙箱、凭证不落地、全链路可审计，安全从第一天就在架构里
- **分阶段克制** — 先把运行时内核做扎实，每次架构升级都用真实使用数据证明其必要性
